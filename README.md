# 🏃 Fitness Tracker (Yandex Practicum Project)

Модуль фитнес-трекера на Go с разделением логики по пакетам.  
Проект демонстрирует работу со структурами, методами, эмбеддингом и интерфейсами.

---

## 🚀 Функционал

- 📊 **Расчёт активности**
  - пройденная дистанция
  - средняя скорость
  - потраченные калории

- 🏃 **Тренировки**
  - обработка данных о тренировке (шаги, длительность, тип)
  - формирование отчёта

- 🚶 **Дневная активность**
  - учёт прогулок
  - формирование отчёта

- 👤 **Персональные данные**
  - хранение имени, веса и роста пользователя

- 🔌 **Унифицированная обработка**
  - использование интерфейса для разных типов активности

---

## 🧱 Архитектура

Проект разделён на пакеты, каждый из которых отвечает за свою область:

- **spentenergy** — расчёт дистанции, скорости и калорий  
- **personaldata** — данные пользователя (имя, вес, рост)  
- **trainings** — обработка тренировок  
- **daysteps** — обработка дневной активности  
- **actioninfo** — интерфейс и общая логика обработки  

📌 Такое разделение делает код:
- модульным
- расширяемым
- удобным для тестирования

## 🔌 Интерфейсы

В пакете `actioninfo` определён интерфейс, который объединяет:

- парсинг данных
- формирование информации об активности

Это позволяет использовать один и тот же код для:

- тренировок (`trainings`)
- прогулок (`daysteps`)

---

## 🧩 Особенности реализации

- Используется **эмбеддинг структур** (встраивание personaldata)
- Одинаковые методы реализованы для разных структур
- Полиморфизм через интерфейсы
- Разделение ответственности между пакетами
